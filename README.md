## **서비스명: 뉴스핀**

- **목표:** 금융 및 주식에 관심은 있지만, 복잡한 정보와 진입 장벽 때문에 망설이는 20대 초중반 청년들을 위한 학습 기반 모의 투자 플랫폼. 실제 시장 뉴스를 분석하고 예측하는 경험을 통해 주식 투자의 기본 원리를 익힐 수 있도록 도움.
- **타겟:** 경제 지식이 부족하거나 모호하고, 주식 투자를 시도하기 두려워하는 20대 청년층.

### (1) 뉴스 분석 학습 서비스

이 기능은 주식 시장의 흐름을 이해하고, 정보를 분석하는 능력을 키우는 데 중점.

- **뉴스 분석 퀴즈**: AI가 제공하는 뉴스 기사를 읽고, 해당 기사가 주식 시장에 **호재**인지 **악재**인지 분석하는 퀴즈를 제공합니다. AI의 감성 분석 점수(예: 긍정/부정 점수 1~100)와 해당 점수를 도출한 핵심 문장/맥락을 함께 제공하여 사용자의 판단을 돕고, 올바른 분석법을 학습..
- **이벤트 분석 학습**: 코로나19 확산, 금리 인상 발표 등 주가에 큰 영향을 미치는 특정 **이벤트**들을 재구성하여, 해당 사건이 주식 시장에 어떤 영향을 미쳤는지 학습할 수 있는 콘텐츠를 제공합니다.

### (2) 모의 주식 투자 서비스

학습을 통해 얻은 지식을 실제 투자에 적용해보는 실전 시뮬레이션.

- **기간별 모의 투자**: 2021년 상반기 3개월치와 같이 특정 기간의 실제 뉴스와 이벤트를 기반으로 모의 투자 환경을 조성합니다. 사용자는 해당 기간 동안 공개되는 뉴스를 분석하며 주식을 사고팔고, 포트폴리오를 관리합니다.
- **AI 기반 투자 결과 분석**: 모의 투자 종료 후, AI가 사용자의 투자 방식을 분석해 'AI 투자 보고서'를 제공.
    - **잘못된 판단 분석**: 특정 이벤트(예: 제약회사 호재 뉴스)를 잘못 분석하여 손실을 본 경우, "해당 뉴스를 오판하여 제약회사 주식을 매도함으로써 기회를 놓쳤습니다"와 같이 구체적인 피드백을 제공. 이는 사용자가 자신의 실수를 명확하게 인지하고 다음 투자에 반영할 수 있도록 도움.

🚀 유스케이스: 로그인 (UC-001)

| **항목** | **내용** |
| --- | --- |
| **유스케이스 ID** | UC-001 |
| **유스케이스 이름** | 로그인 (User Authentication) |
| **액터 (Actor)** | 사용자 (User) |
| **목표** | 사용자가 유효한 자격 증명으로 시스템에 인증하고 서비스 메인 화면으로 이동한다. |
| **선행 조건** | 사용자가 NewsPin 로그인 화면에 접속한 상태이다. |
| **종료 조건** | **[성공]** 사용자가 서비스 메인 화면으로 이동하고 인증된 세션이 생성된다.
**[실패]** 사용자가 로그인 화면에 남아있으며 오류 메시지를 확인한다. |

| **흐름** | **액터 행동 (User Action)** | **시스템 반응 (System Response)** |
| --- | --- | --- |
| **기본 흐름** | 1. 아이디와 비밀번호를 입력 필드에 입력한다. | 시스템은 아이디와 비밀번호 입력 필드, "투자 여정 시작하기" 버튼을 제공한다. |
| **(성공)** | 2. **"투자 여정 시작하기"** 버튼을 클릭한다. | 시스템은 입력된 자격 증명을 인증 서버로 전송하고 유효성을 검증한다. |
|  |  | 3. 유효성 검증에 성공하면, 시스템은 사용자를 **서비스 메인 화면**으로 이동시킨다. |
| **대안 흐름** | **A1: 유효하지 않은 자격 증명** | 3. 유효성 검증에 실패하면, 시스템은 사용자에게 **오류 메시지**를 표시한다 (예: "아이디 또는 비밀번호를 잘못 입력하셨습니다."). |
| **보조 기능** | **회원가입 / 비밀번호 찾기** 글자를 클릭한다. | 시스템은 **호버 효과**를 적용하고 **클릭 시**, 해당 페이지로 이동시킨다. |

🔑 유스케이스: 비밀번호 찾기 (UC-003)

| **항목** | **내용** |
| --- | --- |
| **유스케이스 ID** | UC-003 |
| **유스케이스 이름** | 비밀번호 찾기 (Password Recovery) |
| **액터 (Actor)** | 사용자 (User) |
| **목표** | 사용자가 인증을 통해 기존 비밀번호를 재설정하여 계정에 다시 접근한다. |
| **선행 조건** | 사용자가 NewsPin 비밀번호 찾기 화면에 접속한 상태이다. |
| **종료 조건** | **[성공]** 사용자의 비밀번호가 재설정되고, 임시 비밀번호가 이메일로 전송된다.
**[실패]** 인증 정보 불일치 메시지가 표시된다. |

| **흐름** | **액터 행동 (User Action)** | **시스템 반응 (System Response)** |
| --- | --- | --- |
| **기본 흐름** | 1. 계정에 등록된 **아이디**와 **이메일**을 입력 필드에 입력한다. | 시스템은 아이디 및 이메일 입력 필드, "이메일로 인증 코드 전송" 버튼을 제공한다. |
| **(성공)** | 2. **"이메일로 인증 코드 전송"** 버튼을 클릭한다. | 시스템은 입력된 아이디와 이메일 주소가 시스템에 등록된 정보와 일치하는지 확인한다. |
|  |  | 3. 인증 정보가 일치하면, 시스템은 **기존 비밀번호를 초기화**하고 **무작위 임시 비밀번호**를 기재된 이메일로 전송한다. |
|  | 4. (이후 단계) 사용자는 이메일로 전송된 **무작위 비밀번호**로 **로그인이 가능**하며 필요 시, 변경이 가능하다. | 시스템은 인증 코드를 검증하고 로그인 및 비밀번호 재설정 페이지로 안내한다. |
| **대안 흐름** | **A1: 정보 불일치** | 3. 입력된 아이디와 이메일이 시스템의 정보와 일치하지 않으면, 오류 메시지를 표시한다 (예: "입력하신 정보와 일치하는 계정이 없습니다."). |
| **보조 기능** | 좌측 상단의 **뒤로가기 화살표** 버튼 ← ****을 클릭한다. | 시스템은 현재 화면에서 **작성한 내용을 취소**하고 **로그인 화면**으로 이동시킨다. |

📝 유스케이스: 회원가입 (UC-002)

| **항목** | **내용** |
| --- | --- |
| **유스케이스 ID** | UC-002 |
| **유스케이스 이름** | 회원가입 (User Registration) |
| **액터 (Actor)** | 사용자 (User) |
| **목표** | 사용자가 새로운 계정을 생성하여 서비스에 접근할 수 있는 권한을 얻는다. |
| **선행 조건** | 사용자가 NewsPin 회원가입 화면에 접속한 상태이다. |
| **종료 조건** | **[성공]** 사용자의 계정이 생성되고 로그인 상태로 메인 화면에 진입한다.
**[실패]** 오류 메시지가 표시되고 사용자는 회원가입 화면에 남는다. |

| **흐름** | **액터 행동 (User Action)** | **시스템 반응 (System Response)** |
| --- | --- | --- |
| **기본 흐름** | 1. 아이디, 비밀번호, 비밀번호 확인, 이메일을 각 필드에 입력한다. | 시스템은 각 필드의 유효성 검사를 실시간으로 수행한다. (필수 입력, 형식 확인 등) |
| **(성공)** | 2. 비밀번호 입력 시, **눈 모양 아이콘**을 클릭하여 **입력한 내용을 확인**하거나 **재클릭시 가린다**. | 시스템은 아이콘 클릭에 따라 비밀번호 필드의 텍스트 표시/가리기를 토글(Toggle)한다. |
|  | 3. **"가입 완료! 로그인 후 투자 감각을 깨워보세요."** 버튼을 클릭한다. | 시스템은 입력된 정보를 저장하고 계정을 생성한다. |
|  |  | 4. 회원가입이 성공하면, 시스템은 로그인 계정을 생성 후 사용자를 **로그인 화면**으로 이동시킨다. |
| **대안 흐름** | **A1: 유효성 검증 실패** | 3. 필수 입력 필드가 누락되거나, 비밀번호와 비밀번호 확인이 일치하지 않는 등 유효성 검증에 실패하면, 오류 메시지를 표시한다. |
| **보조 기능** | 좌측 상단의 **뒤로가기 화살표** 버튼 **←**을 클릭한다. | 시스템은 현재 화면에서 **작성한 내용을 취소**하고 **로그인 화면**으로 이동시킨다. |
